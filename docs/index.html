What is this?

Stompbox turns your Mac into a guitar pedalboard. Plug in a mic or an instrument, wire up some effects, and tweak everything with MIDI controllers. It all runs inside a terminal UI that shows your signal chain, live VU meters, and incoming MIDI data at a glance.

Under the hood, it uses Spotify's pedalboard library for plugin hosting, which means you get access to 19 built-in effects plus any VST3 or Audio Unit plugin on your system. Virtual instrument plugins work too, generating audio straight from MIDI input.

The whole thing is configured with a single YAML file. No DAW required. No GUI windows stealing focus. Just audio in, effects, audio out, and a TUI that stays out of your way.

Getting started

Install stompbox with pip, then initialize a project directory. This creates a stompbox.yml config, a chains/ folder with a few presets, and a recordings/ directory for captured audio.

pip install stompbox
stompbox init
stompbox run

That's it. The TUI launches, your default audio device opens, and you're processing audio through whatever chain is defined in your config. If you want to see what audio and MIDI devices are available on your system, there's a command for that too.

stompbox devices

Configuration

Everything lives in stompbox.yml. You pick your audio devices, set your sample rate and buffer size, point at a MIDI input, and define your signal chain as an ordered list of plugins. Each plugin can have parameters dialed in and MIDI CC mappings assigned right in the config.

audio:
  input: MacBook Pro Microphone
  output: BlackHole 2ch
  sample_rate: 44100
  buffer_size: 512

midi:
  input: IAC Driver Bus 1

chain:
  - plugin: NoiseGate
    params:
      threshold_db: -40
  - plugin: Compressor
    params:
      threshold_db: -20
      ratio: 3.0
    midi:
      cc:
        74: threshold_db
  - plugin: Reverb
    params:
      room_size: 0.5
      wet_level: 0.2

For external plugins, swap plugin for path and point it at a .vst3 or .component file. Set audio.input to null for instrument plugins that generate audio from MIDI rather than processing an input signal.

audio:
  input: null  # instrument mode
  output: MacBook Pro Speakers

chain:
  - path: /Library/Audio/Plug-Ins/VST3/MyInstrument.vst3

Built-in plugins

Stompbox ships with 19 built-in effects from pedalboard. These load instantly without any external files. You can mix and match them freely in your chain, and they all support real-time parameter control via MIDI CC.

Chain presets

Drop YAML files in the chains/ directory and switch between them on the fly using keyboard shortcuts or MIDI program changes. Each preset is just the chain portion of the config, so you can build up a library of tones and swap them without restarting.

# chains/ambient.yml
chain:
  - plugin: Chorus
    params: { rate_hz: 0.5, depth: 0.4, mix: 0.3 }
  - plugin: Delay
    params: { delay_seconds: 0.35, feedback: 0.4 }
  - plugin: Reverb
    params: { room_size: 0.9, wet_level: 0.5 }

Map MIDI program changes to chain files in your config, and your foot controller becomes a preset switcher.

midi:
  input: IAC Driver Bus 1
  program_change:
    0: chains/clean.yml
    1: chains/drive.yml
    2: chains/ambient.yml

Keyboard shortcuts

The TUI responds to single keypresses for everything you need during a session. Bypass individual pedals by number, kill the whole chain at once, or flip through your presets without ever touching a mouse.

1through 9 — toggle bypass on that pedal SPCmaster bypass — all pedals on or off rreset peak meters [previous chain preset ]next chain preset qquit

How it works

Stompbox runs the audio engine in a separate process. Native VST3 and AU plugins love to write debug output to stdout from background threads, which would corrupt the terminal UI. By isolating the engine in a child process with its stdout and stderr redirected to /dev/null, the TUI gets a clean terminal to paint on. The two processes talk over a pipe at 30 frames per second, so the meters and chain state you see are never more than one frame stale.

Audio flows through PortAudio via sounddevice, MIDI comes in through macOS CoreMIDI (direct ctypes, no compiled dependencies), and plugins are hosted by pedalboard's Juce-based infrastructure. The TUI is built with Textual, rendering at 30fps for meters and 10fps for the chain view. The whole architecture is lock-free where it counts: audio callbacks, meter reads, and MIDI dispatch all rely on Python's GIL for atomic operations instead of explicit locks.

Headless mode skips the subprocess isolation entirely and runs everything in a single process with a minimal status line. Useful for recording or running as a background audio processor.
